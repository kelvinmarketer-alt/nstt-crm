File báo giá PDF/HTML gửi khách
A3Lập KH

B Luồng đơn hàng (xương sống)

Mô hình demand-driven: có đơn mới mới gom & đặt NCC, hạn chế tồn kho.
B1Tạo đơn
Sale
Việc cần làm
Nhận đặt hàng → tạo đơn. Lưu đơn xong → hệ thống ra ngay PHIẾU BÁO HÀNG (in / gửi Telegram cho khách & kho).
Tính năng dùng
Đơn hàng → + Tạo đơn mới hoặc 📷 Tạo đơn từ ảnh → tự sinh 📄 Phiếu báo hàng
Đầu ra
Đơn "Mới" + phiếu báo hàng
B2Đơn web
Sale
Việc cần làm
Khách tự đặt trên website → đơn vào hộp chờ duyệt → Sale kiểm tra & duyệt thành đơn chính thức.
Tính năng dùng
Đơn hàng → nút đỏ 🛒 Đơn web chờ duyệt → Duyệt / Từ chối
Đầu ra
Đơn web → đơn chính thức
B3Gom
Kho
Việc cần làm
Tick các đơn cùng đợt giao → tạo phiên gom → hệ thống cộng tổng sản lượng từng mặt hàng.
Tính năng dùng
NCC·Nhập·Gom hàng → tab Gom hàng → tick đơn → 🌾 Tạo phiên gom

Phiên gom (tổng SL theo SP)
B4Đặt NCC
Kho / Mua hàng
Việc cần làm
Trong phiên gom: gán NCC cho từng mặt hàng → in phiếu yêu cầu / copy gửi Zalo cho NCC.
Tính năng dùng
Mở phiên → ô NCC (gõ tên, gợi ý) → 🖨 In phiếu / 📋 Copy Zalo
Đầu ra
Phiếu đặt hàng gửi từng NCC
B5Nhận
Kho
Việc cần làm
NCC giao → tạo Phiếu nhập, bấm "Đã nhận" để cộng kho + cập giá nhập + công nợ NCC. Thiếu hàng → nhập "giao thực" → hệ thống cắt theo ưu tiên đơn đặt trước + báo Sale.
Tính năng dùng
tab Phiếu nhập → ✓ Nhận · phiên gom → nhập "Giao thực" → ✅ Chốt & phân bổ + báo Sale
Đầu ra
Tồn +; công nợ NCC; báo thiếu (TG)
B6Xuất kho
Kho
Việc cần làm
Đơn đã chốt hàng → in phiếu xuất kho (kèm ca/giờ giao) → bàn giao shipper.
Tính năng dùng
Gom hàng → bước ③ Xuất kho → 🖨 Phiếu xuất kho → 🛵 Giao shipper
Đầu ra
Phiếu xuất kho; đơn → "Đang giao"
B7Giao
Shipper
Việc cần làm
Giao hàng. KHÔNG tự gắn "thành công" — phải xác nhận kết quả thực tế: Giao đủ / Khách trả lại / Thiếu.
Tính năng dùng
Đơn hàng → đổi trạng thái Đã giao → popup ✅ Giao đủ / ↩️ Trả hàng / ⚠️ Thiếu (kèm chụp ảnh POD ở Trả hàng)
Đầu ra
Đơn "Đã giao" + thông báo TG
B8Trả hàng
Kho / Sale
Việc cần làm
Nếu trả/thiếu: ghi mặt hàng + số kg → chọn Bỏ (hao hụt) hay Nhập lại kho → lỗi do Vận chuyển/NCC/KH → hoàn tiền (giảm công nợ / chi tiền mặt). Lỗi NCC → tạo khoản đòi NCC.
Tính năng dùng
Trả hàng → + Ghi nhận trả hàng → ✓ Xử lý
Đầu ra
Hoàn tiền/giảm nợ; +tồn; đòi NCC
B9Đối soát
Kế toán
Việc cần làm
Ghi nhận COD/đã thu, cập nhật công nợ KH, đối soát sổ quỹ & ngân hàng, xuất hóa đơn nếu cần.
Tính năng dùng
Tài chính → Sổ quỹ · Công nợ · Hóa đơn
Đầu ra
Dòng tiền + công nợ chuẩn

D Dữ liệu tự liên thông giữa các phòng

Một thao tác kéo theo cập nhật tự động ở phòng khác — chỗ dễ "lủng củng" nhất, cần khớp với quy trình thực tế.
→
Khi Sale chốt đơn
Gửi phiếu báo hàng + đơn vào hàng đợi gom
Tự động
Sale → Kho nhận tín hiệu cần gom
→
Khi Kho "Đã nhận" phiếu nhập
Cộng tồn + cập giá nhập + công nợ NCC
Tự động
Kho → Kế toán (công nợ phải trả) + Bảng giá (giá nhập mới)
→
Khi xác nhận "Đã giao"
Ghi doanh thu + công nợ KH + trừ tồn
Tự động
Shipper → Kế toán + Kho
→
Khi trả hàng (nhập lại kho)
Cộng lại tồn + giảm công nợ/chi tiền + đòi NCC
Tự động
Kho → Kế toán + NCC
→
Khi Sếp đổi giá Gốc (= giá nhập +%)
Áp cho toàn hệ thống: đơn, website, các nhóm giá
Tự động
Ban GĐ → tất cả phòng
→
Khi HR tính lương
Gửi CFO/Sếp duyệt → chi lương vào sổ quỹ
Tự động
HR → CFO → Kế toán
